# Code of Conduct

ShioriCode's community spaces should be direct, useful, and respectful.

## Expected Behavior

- Be clear and specific.
- Assume maintainers and contributors are balancing real constraints.
- Keep criticism focused on code, behavior, docs, and product tradeoffs.
- Respect maintainer decisions about scope and timing.
- Do not post secrets, private logs, or sensitive user data.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Repeatedly pushing closed decisions.
- Posting exploit details before maintainers have had time to respond.
- Derailing issues or pull requests with unrelated arguments.
- Spam, low-effort promotion, or automated noise.

## Enforcement

Maintainers may edit, hide, or delete comments; close issues or pull requests; restrict participation; or block users when needed to protect the project.

If you see a problem, contact the maintainers privately with links and context.
