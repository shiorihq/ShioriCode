# Support

ShioriCode is early software. Support is best-effort and primarily happens through GitHub issues.

## Where To Ask

- Use bug reports for reproducible broken behavior.
- Use feature requests for scoped product improvements.
- Use discussions or issues for setup questions if the answer should become public documentation.

## Before Opening An Issue

- Search existing issues.
- Try the latest main branch or latest release.
- Include your OS, Bun version, Node version, ShioriCode version or commit, and provider.
- Include concise logs with secrets redacted.

## What We Prioritize

- Security issues.
- Data loss or corrupted session state.
- Provider session failures.
- Reconnect and streaming reliability.
- Build, release, and installation failures.

General usage questions and broad feature requests may take longer.
